# Token 與有效性的平衡點
## 每次任務都會出現的內容
只放在 AGENTS.md: 
- 安全限制
- Git 規範
- 測試原則
- 輸出格式
- 套件政策

## 每次任務才寫的內容
只需要寫:
- 這次要完成什麼
- 使用者可觀察到的行為
- 本次特有的限制
- 如何驗證
- 是否允許 Commit／Push

## 不應該反覆提供:
- 完整專案背景
- 完整聊天紀錄
- 所有 Terminal 輸出
- AGENTS.md 已經存在的規則
- 每一條 Git 或 Shell 操作步驟
- 與本次任務無關的檔案內容

## 小任務不必做的事
- 不必先進入 Plan mode。
- 不必要求 Codex 掃描整個 Repository。
- 不必再呼叫一次 AI Review。
- 不必建立 Pull Request。
- 不必保存每一則 Prompt。
- 不必列出所有 Git 指令。
- 不必要求長篇解釋。
